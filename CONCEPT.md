# コンセプト

> バイク車種別の通行制限を考慮したGoogleマップ連携アプリ

## 目的

排気量によって通れない道路があるバイクライダーのために、車種に応じた通行制限を考慮したルート案内を提供する。

Googleマップは車種別の通行制限を考慮しないため、原付・125ccライダーが自動車専用道路に誘導されるなどの問題がある。このアプリはツーリング前の事前確認ツールとして、その問題を「Googleマップよりマシ」なレベルで解消することを目標とする。

## ポジショニング

商用の「ツーリングサポーター by NAVITIME」は、警察提供データと自社ルーティングエンジンで車種別案内を実現している。個人開発で同等のものは作れない。

このアプリの強みは「無料・OSS・コミュニティ駆動」。完璧なルーティングではなく、Googleマップに車種別の通行禁止情報を加えるハイブリッド方式を選んだ。

## 主要な設計決定

### アプリ形態

**Webアプリ一択。** 主な利用シーンは「ツーリング前の下調べ」であり、ネイティブアプリの強み（オフライン地図・バックグラウンド位置取得）が不要なため。

### 車種区分

3区分に固定：原付一種（50cc・新基準原付）/ 原付二種（125cc以下）/ 普通二輪以上（126cc〜）。

avoid 設定はユーザーには見せず、車種選択で自動的に決まる。

### ルーティングアプローチ

**ルーティング自体はGoogle Routes APIに委ねる。** 車種別の禁止区間情報だけ自前のDBで管理し、ルートのpolylineと空間交差判定して警告を出す方式。

### データソース

**OSMをメインソースとして使用。**

| OSMタグ | 意味 | 対象車種 |
|---|---|---|
| `motorcycle=no` | 二輪通行禁止 | 全車種 |
| `moped=no` | 原付通行禁止 | 原付一種 |
| `motorroad=yes` | 自動車専用道路 | 原付一種・二種 |

高速道路（highway=motorway）は Routes API の `avoid: highways` で自動回避されるため、DBへの個別登録は不要。

ユーザー投稿（写真付き・承認制）でOSMに登録されていない区間を補完する。

### データ投稿の粒度

**区間方式（始点・終点）。** ポイント方式では精度が低く、OSMリンクID方式では一般ユーザーに難しいため。
